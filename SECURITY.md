# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

We only provide security fixes for the current release version of Novellum.

## Reporting a Vulnerability

If you discover a security vulnerability in Novellum, please report it responsibly by emailing:

**security@gibsondevhouse.com**

Please do **not** open a public GitHub issue for security vulnerabilities.

Include the following in your report:

- A description of the vulnerability and its potential impact
- Steps to reproduce the issue
- Any proof-of-concept code or screenshots (if applicable)
- The version of Novellum you are using

## Response Timeline

| Stage                    | Timeline        |
| ------------------------ | --------------- |
| Initial acknowledgement  | Within 3 days   |
| Assessment and triage    | Within 7 days   |
| Fix or mitigation        | Within 30 days  |
| Public disclosure        | After fix ships |

We will keep you informed throughout the process and credit you in the release notes if you wish.

## Out of Scope

The following are considered out of scope for security reports:

- Vulnerabilities in third-party dependencies (report these to the upstream project)
- Social engineering attacks
- Physical access attacks
- Issues requiring a fully compromised operating system
- Denial-of-service attacks against the local application
- Theoretical vulnerabilities with no practical exploitation vector
