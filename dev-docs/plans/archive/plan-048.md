## Objective

Build the deterministic large-novel test fixture and timing harness named in the roadmap
under "Planned but not started." This gives the project measurable performance baselines
and regression guards before public release, and provides a benchmark surface for
evaluating the impact of the Summary Agent (plan-044) and semantic indexing (plan-046)
on token and load times.

## Scope

**In scope:**

- `tests/fixtures/large-novel/` — deterministic SQLite seed with:
  - 100 chapters across 3 arcs
  - 500 scenes
  - ~100,000 words of generated prose content
  - 50 worldbuilding entities (characters, realms, factions, lore entries)
- Vitest timing tests measuring:
  - Scene list load time (p50, p95)
  - Chapter navigation transition time
  - `ContextEngine` context build time for a 20-scene window
  - `ContinuityAgent` invocation time (with and without stored summaries)
  - FTS5 full-text search query time (once plan-046 is merged)
- CI integration: timing tests run in a dedicated `pnpm test:perf` script; regressions
  are flagged but do not block the main `pnpm test` gate.
- Documented baseline numbers in `dev-docs/05-workflow/performance-baseline.md`.

**Out of scope:**

- UI-level performance profiling (Playwright paint timings).
- Memory profiling.
- Distributed / concurrent load testing.

## Risks & Mitigations

| Risk | Likelihood | Mitigation |
| ---- | ---------- | ---------- |
| Generated prose content is non-deterministic if seeded by an LLM | medium | Use a deterministic lorem-style generator (e.g., seeded Markov or fixed seed faker) — not a live AI call |
| CI machine variance makes timing thresholds flaky | medium | Set thresholds at 3× the local p95; document that numbers are guidance, not hard SLOs |

## Notes

This plan is referenced by plans 044 (Summary Agent) and 046 (semantic indexing) as
their benchmark target. The fixture should be checked into the repo as a binary SQLite
file or generated by a deterministic seed script that produces the same output on every
run. The fixture must not depend on live AI calls.
