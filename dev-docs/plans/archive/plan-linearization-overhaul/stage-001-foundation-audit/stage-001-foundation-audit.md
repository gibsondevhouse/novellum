# Stage 001: Foundation & Audit

## Goal
Establish a baseline of existing UI patterns and identify deviations from the token system.

## Phases
### Phase 001: Inventory
Audit all current UI surfaces (Sidebar, Cards, Buttons) and document their current implementation state.

### Phase 002: Token Analysis
Map hardcoded values to existing CSS tokens or propose new tokens in `src/styles/tokens.css`.
