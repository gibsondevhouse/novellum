# Phase 001: App Settings & Onboarding

## Strategy
Build the remaining global infrastructure for user configuration and ensure new users aren't met with a blank, confusing screen.

## Acceptance Criteria
1. Users can manage their OpenRouter API key and default models from the UI.
2. A "Getting Started" guide or onboarding modal appears for new installations.

## Edge Cases
- Distinguishing between server-side .env keys and user-provided UI keys.
