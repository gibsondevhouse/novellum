## Phase-001: Storyboard and Workflow Surfaces

## Goal

Polish the high-use workflow routes into release-ready creative rooms without compromising productivity.

## Implementation Strategy

Use the cinematic system as workflow support, not decoration. The outline and editor should prioritize hierarchy scan speed, typing comfort, contextual cues, and recovery from errors. Nova, Images, Settings, and Styles should use the same shell/primitives so they no longer feel disconnected.

## Acceptance Criteria

- [ ] `/outline` communicates story hierarchy and selected detail without hiding structure.
- [ ] `/editor` keeps manuscript focus, save status, scene context, and AI tools accessible.
- [ ] `/nova` has polished empty and conversation states.
- [ ] `/images` has asset gallery, inspection, assignment, and destructive confirmation states.
- [ ] `/settings`, `/settings/migrate`, and `/styles` share production configuration patterns.

## Edge Cases

- Dense outlines and long scenes must remain usable; avoid excessive card nesting or oversized decorative panels.
