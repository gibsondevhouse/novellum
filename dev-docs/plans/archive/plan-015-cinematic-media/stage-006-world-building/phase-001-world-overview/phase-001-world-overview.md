## Implementation Strategy

Preserve the current semantic domains while improving orientation and visual hierarchy. Use shared primitives for domain tiles, entity posters, dossier shells, continuity triage, and empty/future-state surfaces.

## Acceptance Criteria

- [ ] Personae, Atlas, Archive, Threads, and Chronicles each have distinct but cohesive domain treatment.
- [ ] Entity collection pages use the correct visual pattern for the workflow.
- [ ] Entity detail pages remain productive forms while gaining visual anchors and structured sections.
- [ ] Continuity issues are grouped, readable, actionable, and accessible.
- [ ] Redirect and future-state pages are polished.

## Edge Cases

- Large lore/archive collections may need a dense mode; dense mode must still use the cinematic shell and tokenized surfaces.
