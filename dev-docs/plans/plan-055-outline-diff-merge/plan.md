## Objective

Evolve the outline materialization route from an all-or-nothing overwrite (which currently returns `outline_conflict` and blocks accept when hierarchy rows exist) into a granular outline diff and selective merge engine. This will allow authors to modify specific acts, chapters, or scenes generated by the AI without wiping out their existing manual changes.

## Scope

**In scope:**
- An outline diff engine on the server to compare a generated `OutlineDraft` checkpoint against the existing `arcs`, `acts`, `chapters`, and `scenes` table hierarchy.
- A visual selective merge screen rendering a tree of changes (insertions, updates, deletions) with per-node checkboxes.
- Modifying the accept route (`POST /api/outline/checkpoints/{checkpointId}/accept`) to accept a payload specifying which subset of nodes to materialize.
- Conflict checks to protect manually written scene content from being silently overwritten during an outline merge.

**Out of scope:**
- Automatic prose generation during merge (handled by scene-drafting loops).
- Merging characters or worldbuilding entities (handled by `vibe-worldbuild` diff/merge).

## Risks & Mitigations
- **Risk:** Authors might accidentally delete manual scenes by unchecking them or accepting deletions.
- **Mitigation:** Force an explicit "destructiveness alert" modal when accepting outline changes that result in deleting existing chapters or scenes, and stage deleted scenes into `scene_snapshots` as automatic backups.
