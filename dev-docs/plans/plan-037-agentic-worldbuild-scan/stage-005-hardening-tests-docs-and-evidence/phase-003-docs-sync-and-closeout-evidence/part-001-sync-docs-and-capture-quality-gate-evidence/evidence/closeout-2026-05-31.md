## Regression Proof — No Pre-Accept Canon Writes

`tests/world-building/worldbuild-proposal-canon-safety.test.ts` proves:
- After `seedProposal(db)` (status = `pending_review`), all 5 canon tables are empty for the project.
- Successful `acceptProposalAtomically` writes exactly 1 row to the correct table.
- A failed accept (missing name field) leaves all tables empty and the proposal still `pending_review`.
- An unrecognized `categoryId` throws `projection_failed` and leaves tables empty.
- Double-accept is idempotent at the store level (exactly 1 canon row after 2 accept calls — second is blocked by `invalid_transition`).
